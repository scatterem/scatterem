# Security Policy

## Supported versions

The public code release has not happened yet, so there are no stable public versions to support at this time.

Once the first public release is published, this policy will be updated with supported version ranges and response expectations.

## Reporting a vulnerability

Please do not report security issues in public GitHub issues.

Instead, email philipp.pelz@yahoo.de with:

- a clear description of the issue
- the affected component or workflow
- reproduction steps or a proof of concept, if available
- any potential impact you have identified

We will aim to acknowledge receipt within a reasonable time and coordinate next steps privately.

## Scope during the pre-release phase

Because the main implementation is not yet public, some reports may be hard to validate until the code release. Even so, responsible disclosure is appreciated, especially for:

- packaging or supply-chain concerns
- CI or release workflow problems
- dependency risks in the public scaffold
- documentation that could lead users to unsafe assumptions
