# Support

## Where to ask what

Use GitHub Issues for:

- bug reports about the public repository scaffold
- documentation problems
- feature requests
- reproducibility expectations for the future public release

Use discussions or issue threads for:

- questions about intended workflows
- requests for examples or tutorials
- interoperability questions with adjacent tools or datasets

Use the security contact in [SECURITY.md](SECURITY.md) for:

- vulnerabilities
- supply-chain concerns
- sensitive reports that should not be public

## What helps maintainers most

The most useful early feedback usually includes:

- the scientific workflow you want to run
- expected input data types and scales
- hardware assumptions
- output expectations and evaluation criteria
- blockers that would prevent adoption in your lab or pipeline

## Response expectations

This project is in an active pre-release phase. Maintainer time may be uneven around publication milestones, but thoughtful issues and well-scoped suggestions are welcome.
