# Discussion Guidance

If GitHub Discussions is enabled, use it to separate open-ended conversation from actionable repository work.

## Good fit for Discussions

- workflow questions
- requests for examples or tutorials
- research use cases
- integration ideas
- community feedback on roadmap direction

## Better fit for Issues

- broken documentation
- packaging or CI failures
- specific feature requests with clear outcomes
- release blockers

## Suggested discussion categories

- Announcements
- Q&A
- Ideas
- Show and tell
- Reproducibility and datasets

## Moderator guidance

- move actionable bug reports into issues
- keep vague questions open long enough to gather useful context
- summarize recurring themes back into docs or roadmap updates
