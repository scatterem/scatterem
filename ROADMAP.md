# Roadmap

This roadmap focuses on making the first public release credible, reproducible, and contributor-friendly.

## Phase 0: Pre-publication scaffold

Status: in progress

Goals:

- establish repository governance and community expectations
- publish contribution guidance and issue templates
- define project scope and release criteria
- gather early user feedback without exposing embargoed code

Exit criteria:

- public repository accurately reflects the pre-release state
- issue intake and contributor workflows are in place
- release notes, citation metadata, and support policy exist

## Phase 1: First public code release

Goals:

- publish the initial supported codebase
- document the install path and supported environments
- provide a small set of reproducible examples
- add user-facing tests for public workflows

Exit criteria:

- package can be installed from source reliably
- examples run with documented dependencies
- public API surface is identified
- known limitations are documented

## Phase 2: Reproducibility and benchmarks

Goals:

- add tutorials tied to concrete use cases
- publish benchmark configurations and baseline comparisons
- document dataset assumptions and preprocessing steps
- improve test coverage for core numerical components

Exit criteria:

- at least one end-to-end example is fully reproducible
- benchmark methodology is documented
- regression checks cover core workflows

## Phase 3: Community growth

Goals:

- expand examples and tutorials
- clarify extension points for external contributors
- improve issue triage and release cadence
- document compatibility across Python, PyTorch, CUDA, and Warp versions

Exit criteria:

- contributors can identify beginner-friendly areas
- release notes consistently describe user-facing changes
- support expectations are documented and maintained
